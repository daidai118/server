## M4 形成打怪闭环

### 范围

- 怪物刷新
- 普攻 / 受伤 / 死亡 / 重生
- 掉落 / 拾取 / 背包 / 装备
- 商店买卖
- 技能基础版

### 验收

- 完整形成“打怪 -> 掉落 -> 拾取 -> 装备”闭环
- 重登后物品和数值不丢
- 经济类操作都有流水

## M5 形成可运营版本

### 范围

- 后台登录与 RBAC
- 账号/角色管理
- 公告
- GM 审计
- 好友/组队/公会等基础社交持久化

### 验收

- 高风险 GM 动作可追责
- 基础运营动作无需直接上 DB
- 社交关系与仓库不再是纯内存态

## 当前建议

在原计划基础上增加一条硬约束：

- **所有 transport 假设必须先拿客户端源码或抓包证据验证。**

原因：当前参考仓库把多处传输行为简化成“文本 + XOR”，但客户端源码已经显示存在 `rnPacket` 帧和 `SEED` 加密实现，不能再按 README 描述直接写主实现。
